Mixed Models with lme4

Stata's xtmixed models can be run using the lmer() function from the lme4 package, no major estimation differences between the two observed so far.

R reports t values for parameter estimates instead of z values, from looking at this in the past I think the author of the lme4 package has strong opinions about whether it's appropriate to test parameter estimates for "significance".

GEE Models

Two packages in R: gee and geepack, not sure what the exact differences are but gee appears to reproduce the Stata results.

  • May have to make the tolerance smaller, R by default has a larger tolerance value so runs less iterations
  • Not sure what the equivalent of Stata's nmp option is.

In R:

# gee matches Stata (with nmp on) as long as you make the tolerance smaller than the
#   default of 0.001: with default it only does 3 iterations, decreasing tolerance
#   increases iterations to 5 and results match Stata
hanley_gee = gee(y ~ 1, id = cluster, 
                 data = hanley, 
                 corstr = 'exchangeable', 
                 tol=0.000001)
